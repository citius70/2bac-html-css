¿Qué es la semántica en HTML5?

La semántica se refiere al significado del contenido. En HTML5, los elementos semánticos describen claramente su propósito tanto para el navegador como para el desarrollador.

En lugar de usar solo <div> para todo, HTML5 nos ofrece elementos específicos para cada parte de una página web.

Elementos Semánticos Principales

<header>

Representa la cabecera de una página o sección. Suele contener el logo, título y navegación principal.

<nav>

Contiene los enlaces de navegación principales del sitio.

<main>

Define el contenido principal de la página. Solo debe haber uno por documento.

<section>

Agrupa contenido temáticamente relacionado.

<article>

Representa contenido independiente, como un post de blog o noticia.

<aside>

Contiene contenido relacionado indirectamente, como barras laterales.

<footer>

Representa el pie de página con información de contacto, enlaces, etc.

Ejemplo de Estructura

Así se vería el código HTML de una página semántica:

<body>
  <header>
    <h1>Mi Sitio Web</h1>
    <nav>...</nav>
  </header>
  
  <main>
    <section>
      <h2>Sección 1</h2>
      <p>Contenido...</p>
    </section>
    
    <article>
      <h2>Artículo</h2>
      <p>Contenido...</p>
    </article>
    
    <aside>
      <h3>Información adicional</h3>
      <p>Contenido relacionado...</p>
    </aside>
  </main>
  
  <footer>
    <p>Pie de página</p>
  </footer>
</body>

Ventajas de Usar HTML5 Semántico

  • Mejor SEO: Los motores de búsqueda entienden mejor tu contenido
  • Accesibilidad: Los lectores de pantalla pueden navegar más fácilmente
  • Mantenimiento: El código es más claro y fácil de entender
  • Compatibilidad: Mejor integración con otras tecnologías

¿Sabías que?

HTML5 también incluye elementos semánticos como <figure>, <figcaption>, <time> y <mark> para contenido más específico.
