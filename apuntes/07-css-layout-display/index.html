    8. CSS Display¶

    La propiedad display en CSS es la que decide cómo se muestran y distribuyen los elementos en una página web.

    Gracias a display, puedes controlar si un elemento ocupa toda una línea, si se coloca junto a otros, si se adapta a un diseño flexible o si directamente desaparece de la vista.
    Es la base para crear tanto estructuras simples (como menús o párrafos) como diseños complejos (galerías, layouts adaptables, etc.).

    Display

    La propiedad display controla cómo se comporta un elemento en el flujo del documento, es decir, cómo se muestra, ocupa espacio y se relaciona con otros elementos.

    8.1 Sintaxis básica¶

    elemento {
  display: valor;
}
    El elemento puede ser cualquier entidad de HTML: párrafo, título, imagen.... Pero, ¿qué valores puede tomar?

    8.2 Opciones display:¶

    La propiedad display acepta hasta 24 opciones distintas, siendo inline, block y inline-block las más ytilizadas.

    8.3 Valores principales¶

    8.3.1 Display: inline¶

    Un elemento con display: inline fluye con el texto, ocupa solo el espacio de su contenido y no respeta width ni height aunque se definan.

    Características clave de display: inline:

    • ❌ NO respeta width y height: Puedes definirlos pero serán ignorados completamente
    • ⚠️ Solo respeta margin-left y margin-right: El margen vertical (top/bottom) no funciona
    • ✅ Respeta padding en todas direcciones: Aunque no mueve otros elementos verticalmente
    • ✅ Fluye con el texto naturalmente: Los elementos se alinean horizontalmente como palabras
    • ✅ No rompe la línea: Otros elementos pueden estar a su lado

    El elemento HTML más común para una caja en línea es el elemento , aunque hay otras.

    Elementos HTML con display: inline por defecto:

    • De formato de texto: <span>, <a>, <strong>, <em>, <b>, <i>, <u>, <small>, <sub>, <sup>

    • Interactivos: <button>, <input>, <label>, <select>, <textarea>

    • Multimedia: <img>, <video>, <audio>

    • Otros: <code>, <mark>, <abbr>, <cite>

    Sintaxis: 

    elemento {
    display: inline;
}

    8.3.2 Display: block¶

    Un elemento con display: block ocupa todo el ancho disponible de su contenedor, genera saltos de línea antes y después, y respeta las propiedades de dimensionamiento (width y height).

    Características clave de display: block:

    • ✅ Respeta width y height: Puedes controlar las dimensiones exactas del elemento
    • ✅ Respeta margin en todas direcciones: Top, right, bottom y left funcionan correctamente
    • ✅ Respeta padding en todas direcciones: Todo el espaciado interno funciona
    • ✅ Ocupa toda la línea: El elemento se expande horizontalmente al 100% del contenedor por defecto
    • ✅ Genera saltos de línea: Antes y después del elemento, forzando que esté solo en su línea

    El elemento HTML más común para una caja en bloque es el elemento <div>, aunque hay muchos otros.

    Elementos HTML con display: block por defecto:

    • Estructura: <div>, <section>, <article>, <header>, <footer>, <nav>, <aside>, <main>

    • Contenido: <p>, <h1> a <h6>, <blockquote>, <pre>, <address>

    • Listas: <ul>, <ol>, <li>, <dl>, <dt>, <dd>

    • Formularios: <form>, <fieldset>, <legend>

    Sintaxis: 

    elemento {
    display: block;
}

    display: inline es ideal para texto, enlaces y componentes pequeños que deben ir en línea con el resto del contenido.

    8.3.3 Display: inline-block¶

    Un elemento con display: inline-block se coloca en línea con otros elementos (uno al lado de otro), pero a diferencia del inline clásico, sí respeta width, height, margin y padding en todas las direcciones.

    Características clave de display: inline-block:

    • ✅ Respeta width y height: Puedes definir sus dimensiones exactas
    • ✅ Respeta margin y padding en todas direcciones: Puedes espaciarlo por fuera y dentro como con un bloque
    • ✅ Se comporta como inline para el flujo: Está en la misma línea que otros inline-block (si hay espacio)
    • ✅ NO rompe línea automáticamente: Se alinean horizontalmente mientras haya espacio en el contenedor, si no, bajan a la siguiente línea
    • 🔁 Ideal para menús horizontales, cajas en galería, botones alineados, etc.

    Elementos que suelen usar inline-block de forma habitual:

    • <img> (ya es inline-block por defecto)
    • <button>, <input>, <select>, <textarea> (por defecto son inline, pero admiten dimensiones como inline-block)
    • Personalmente, cualquier <div>, <span>, etc. puede cambiarse a inline-block vía CSS

    Sintaxis:

    elemento {
    display: inline-block;
}
