    9. CSS Position¶

    La propiedad position en CSS permite controlar dónde se sitúa un elemento dentro de la página.
    Con position puedes mover cajas y botones, sacar elementos fuera del flujo normal, fijarlos en la pantalla o hacer que se queden “pegados” al hacer scroll. Es fundamental para crear menús fijos, cabeceras que se mantienen al hacer scroll, efectos flotantes y layouts complejos.
    Dominar position te da mucho poder para diseñar interfaces atractivas y funcionales.

    Position

    La propiedad position controla cómo se posiciona un elemento en la página.
    Es esencial para crear layouts complejos y efectos avanzados.

    Sintaxis:

    La sintaxis básica de la propiedad position en CSS se combina con las propiedades top, right, bottom y left para mover el elemento según el tipo de posición elegido:

    selector {
    position: valor;
    top: desplazamiento-superior;
    right: desplazamiento-derecha;
    bottom: desplazamiento-inferior;
    left: desplazamiento-izquierda;
}
    • position el valor acepta: static (por defecto), relative, absolute, fixed, sticky
    • top, right, bottom, left: solo funcionan si position NO es static
    • Puedes usar solo las direcciones necesarias (por ejemplo, solo top y left para mover hacia abajo y a la derecha)

    Ejemplo práctico: 

    .caja {
    position: absolute;
    top: 50px;
    left: 20px;
}
    Coloca .caja a 50px del borde superior y 20px del izquierdo de su contenedor posicionado.

    9.1 Opciones de position¶

    Valor Descripción
    static Flujo normal (por defecto). top, right, bottom, left no funcionan
    relative Se desplaza desde su posición original. El espacio original se mantiene reservado
    absolute Posicionamiento exacto respecto al padre. Se sale del flujo normal
    fixed Se fija a la ventana. No se mueve al hacer scroll
    sticky Relativo hasta scroll, luego se fija (híbrido entre relative y fixed)

    9.1.1 position:static (por defecto)¶

    Qué hace:
    El elemento sigue el flujo normal del documento, colocándose uno tras otro según el orden de línea de código HTML.

    Comportamiento:
    - ❌ Las propiedades top, right, bottom, left no tienen efecto
    * Estas propiedades desplazarían el elemnto de su posición original, por ello, no tiene lógica que funcionen.
    - ✅ Los márgenes y padding funcionan normalmente.
    - ✅ Es el comportamiento por defecto de todos los elementos (no necesitas escribirlo).
    - ✅ Los elementos se colocan uno tras otro en el flujo normal

    Sintaxis: 

    .elemento1 {position: static;}
.elemento2 {position: static;}
...
.elementon {position: static;}
/* ↓ Todos siguen el flujo natural */

    9.1.2 position: relative¶

    Qué hace:
    Ahora sí puede ser desplazado de su posición original usando las propiedades top, right, bottom, left, permaneciendo en el flujo normal del documento, pero su espacio original se respeta.

    Comportamiento:
    - ✅ Las propiedades top, right, bottom, left tienen efecto
    - Desplazan el elemento desde su posición original, pero el espacio original se mantiene reservado.
    - ✅ Márgenes y padding funcionan normalmente.
    - ✅ Los elementos se colocan uno tras otro en el flujo normal, salvo que algún elemento se desplace visualmente. - ✅ Otros elementos NO se ven afectados (no ocupan el espacio del desplazamiento)
    - ✅ Útil para pequeños ajustes finos y como contenedor para elementos absolute

    Sintaxis: 

    .elemento1 {position: relative;}
.elemento2 {position: relative;}
...
.elementon {position: relative;}
/* ↓ Todos siguen el flujo natural, pero pueden desplazarse visualmente */

    9.1.3 position: absolute¶

    Qué hace:
    El elemento es retirado del flujo normal del documento y se posiciona respecto al primer contenedor padre con position distinto de static (normalmente, relative, absolute, fixed). Si no hay ninguno, se coloca respecto al viewport (pantalla).

    Comportamiento:
    - ✅ Las propiedades top, right, bottom, left tienen efecto
    * Sitúan el elemento en la posición absoluta dentro de su contenedor posicionado (o la ventana si no hay contenedor).
    - ❌ El elemento NO ocupa espacio en el flujo normal; los demás elementos se comportan como si no existiera, pueden quedar debajo o perder su sitio.
    - ✅ Márgenes y padding funcionan normalmente.
    - ✅ Útil para colocar decoraciones, badges, pop-ups, menús flotantes, tooltips…
    - ⚠️ Si el contenedor padre no tiene position: relative (o similar), el elemento se posiciona respecto a la ventana.

    Sintaxis: 

    .elemento {
    position: absolute;
    top: 20px;       /* Sitúa a 20px del borde superior de su contenedor posicionado */
    left: 100px;     /* Sitúa a 100px del lado izquierdo */
}
.padre {
    position: relative; /* Recomendado para que el absolute no salga fuera de la caja */
}

    9.1.4 position: fixed¶

    Qué hace:
    El elemento se saca completamente del flujo normal y se posiciona siempre respecto a la ventana (viewport), no importa el scroll ni el contenedor padre. El elemento “flota” sobre la página y permanece visible en la misma posición aunque el usuario mueva la página hacia arriba o abajo.

    Comportamiento:
    - ✅ Las propiedades top, right, bottom, left tienen efecto
    * Sitúan el elemento en la ventana, fijo sin moverse al hacer scroll. - ❌ El elemento NO ocupa espacio en el flujo normal; otros elementos no lo ven (podrían quedar debajo).
    - ✅ Márgenes y padding funcionan normalmente.
    - ✅ Útil para menús, botones de ayuda, banners importantes, mensajes emergentes, "voler arriba".
    - ⚠️ Muy importante el uso de z-index si hay otros elementos superpuestos.

    Sintaxis: 

    .elemento {
    position: fixed;
    bottom: 20px;
    right: 30px;
    width: 60px;
    height: 60px;
    z-index: 999; /* Asegura que esté encima de otros */
}

    9.1.5 position: sticky¶

    Qué hace:
    El elemento combina el comportamiento de relative y fixed:
    - Al principio, se comporta como relative (flujo normal).
    - Cuando el usuario hace scroll y el elemento llega a una posición definida (top, left, right, bottom), queda “pegado” a esa posición y se comporta como fixed mientras su contenedor lo permite.

    Comportamiento:
    - ✅ Las propiedades top, left, right, bottom funcionan para definir el punto donde se vuelve “pegajoso”.
    - ✅ El elemento ocupa espacio en el flujo, no se saca del documento.
    - ⚠️ Solo es sticky cuando se encuentra dentro del área visible de su contenedor.
    - ✅ Muy útil para menús, encabezados de tabla, avisos que deben estar visibles al hacer scroll.

    Importante:
    - El contenedor padre no debe tener overflow: hidden/auto/scroll para que funcione correctamente. - Soportado en la mayoría de navegadores modernos.

    Sintaxis: 

    .elemento-sticky {
    position: sticky;
    top: 0;
    background: #ffe082;
    border-bottom: 2px solid #bfa500;
}

    9.2 Tabla comparativa de position¶

    Valor ¿Sigue el flujo normal? ¿Ocupado el espacio original? ¿Propiedades top/right/bottom/left? ¿Respecto a qué se posiciona? ¿Caso de uso típico?
    static ✅ Sí ✅ Sí ❌ No - Valor por defecto. Elementos normales
    relative ✅ Sí (pero se mueve) ✅ Sí ✅ Sí (desplazan visualmente) Su posición original Ajustes finos/locales
    absolute ❌ No ❌ No ✅ Sí Primer ancestro posicionado * Badges, pop-ups, decoraciones
    fixed ❌ No ❌ No ✅ Sí Ventana (viewport) Menús/botones fijos al hacer scroll
    sticky ✅ Sí (hasta “pegarse”) ✅ Sí ✅ Sí (definen el “pegado”) Su contenedor desplazable/scrollable Encabezados pegajosos con scroll

    Notas:

    • Primer ancestro posicionado: el elemento padre más cercano con position diferente de static.
    • Todos los valores permiten usar márgenes, padding y borde normalmente.
    • sticky necesita top/left/right/bottom y suficiente scroll para apreciarse.
